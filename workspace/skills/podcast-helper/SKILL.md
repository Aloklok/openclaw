# 播客助手 (Podcast Helper)

你可以帮助用户转录小宇宙播客节目。只有在用户明确要求处理播客时，才按以下流程操作。

## 识别条件

只有当用户**明确表达要转录、下载、提取内容、生成摘要、处理播客**时，才允许触发本技能。

仅仅出现以下任一情况时，**严禁自动触发**：

- 用户消息里只有链接，没有明确要求处理
- 用户只是转发、记录、收藏、提到某个节目
- 用户在讨论配置、日志、网络、报错、技能本身
- 用户发送的不是小宇宙转录请求，而是普通网页/媒体/聊天内容

## 明确触发示例

以下表达才算显式授权：

- “帮我转录这个小宇宙”
- “把这期播客下载并整理一下”
- “给我总结这个播客内容”
- “调用播客转录服务处理这个链接”

如果用户没有明确要求，而只是发来链接或提到播客，先简短确认意图，不要调用任何后台服务。

## 执行步骤

### 1. 提交转录任务

使用 `web_fetch` 工具向宿主机 API 发送 POST 请求：

- **URL**: `http://host.docker.internal:7861/transcribe`
- **方法**: POST
- **Content-Type**: `application/json`
- **Body**: `{"url": "<小宇宙链接>"}`

返回示例：
```json
{"task_id": "a1b2c3d4", "message": "任务已提交", "status": "queued"}
```

**立即回复用户**：「收到！正在帮你下载和转录，大约需要 15-20 分钟，完成后我会通知你。」

### 2. 异步交接（绝不允许大模型死循环轮询！）

提交 POST 请求拿到 `task_id` 后，**严禁大模型自己进入死等或试图每隔 120 秒发送查询请求！** 因为转录需要 20 分钟以上，你的 Session 等待会烧毁极高的 Token。

**正确处理步骤：**
1. 立即用 `run_command` 在 `workspace/podcast_outputs/` 下生成一个名叫 `ACTIVE_<task_id>` 的标记文件（例如 `touch podcast_outputs/ACTIVE`）。
2. 向用户回复：「收到！小宇宙播客已丢入后台 GPU 转录队列，大概需要 15-30 分钟。小龙虾的【心跳巡检系统】会接手全程监控，一旦完成洗稿会立刻微信弹窗推给你，您可以去忙别的了。」
3. **结束本次对话出栈**。

剩下的工作会由 `HEARTBEAT.md` 里的第一条机制（播客转录监控）在每 30 分钟的心跳苏醒时接管，这才是标准的松耦合异步架构。

### 5. 回复用户

**格式模板**：

```
✅ 转录完成！

📌 标题：{title}

📝 摘要：
{summary}

📁 完整文件已保存在 podcast_outputs/{folder_name}/ 目录下，包含：
- TXT 纯文本
- HTML 美化版
- PDF 笔记版
```

## 读取全文

如果用户要求查看全文（例如「给我看全文」「发给我内容」「播客文稿」），从容器内的挂载目录读取：

```
podcast_outputs/{folder_name}/
```

在该目录下找到 `_AI_Refined.txt` 文件（优先）或普通 `.txt` 文件，用 `read` 工具读取并发送。
由于全文可能很长，分段发送，每段不超过 2000 字。

## 重要事项

- **显式授权优先**：没有用户明确指令时，绝对不要调用 `7861`，也不要做探测、健康检查、预热或自动唤醒。
- **不要主动发送全文**：播客动辄数万字，发到聊天窗口会触发字符限制。只发摘要，等用户要求时才发全文。
- **API 不可用**：只有在用户已经明确要求转录后，才允许检查 `http://host.docker.internal:7861/health`。
  - 如果此时 API 不通，只需告知“后台转录服务当前未启动或不可达，暂时无法处理这期播客”，不要自动拉起，不要自动自愈，不要要求用户之外的隐式操作。
- **路径注意**：API 返回的 `folder_name` 是文件夹名称，不是完整路径。在容器内对应的相对路径必须是 `podcast_outputs/{folder_name}/`，使用工具读取时**绝对不要**在前面拼写 `workspace/`。
- **不要使用 curl**：当前工具配置中没有 exec 工具，所有 HTTP 请求必须通过 `web_fetch` 工具完成。
